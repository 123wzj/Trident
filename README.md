# Trident：双路线 CTI 知识图谱构建、协同推理与增量进化框架

## 1. 项目概述

**Trident** 是一个面向网络威胁情报（CTI）的端到端智能分析框架，目标是将异构、噪声化、时效敏感的威胁数据，转化为可计算、可推理、可持续进化的威胁知识图谱（Threat Knowledge Graph）。

项目围绕两条互补路线展开：

- **路线 A（Online）**：OTX 在线情报节点获取 → 节点富化 → 图谱构建 → 双模型协同推理  
- **路线 B（Incremental）**：多智能体深度抽取 → 节点富化 → 图谱增量更新 → 增量模型训练

这两条路线共同解决 CTI 场景中的核心矛盾：  
**“数据持续变化” 与 “模型/图谱长期有效”** 之间的平衡问题。

---

## 2. 设计动机与问题定义

传统 CTI 管道常见痛点：

1. **情报源异构**：报告、告警、IOC、社区情报格式不统一；
2. **语义稀疏**：仅有 IOC 但缺乏攻击阶段、战术意图、组织归因等上下文；
3. **图谱静态化**：一次性构建后难以跟随威胁演化；
4. **模型漂移**：新攻击技战术出现后，旧模型性能衰减；
5. **推理不稳定**：单一模型在高噪声文本上鲁棒性不足。

Trident 对应提出：

- 在线/离线结合的节点生产机制；
- 面向实体-关系-事件的多维富化策略；
- 支持增量写入与冲突消解的图谱演化机制；
- 双模型协同推理与多智能体抽取协作；
- 持续学习（incremental training）闭环。

## 3. 技术路线 A：OTX 在线情报驱动的图谱构建与双模型协同推理

### 3.1 在线节点获取
从 OTX 持续拉取脉冲（Pulse）及相关 IOC，形成原始节点池。  
典型节点类型包括：

- 恶意 IP / 域名 / URL / Hash
- 攻击组织（Threat Actor）
- 恶意家族（Malware Family）
- 漏洞（CVE）
- 工具与基础设施（Tooling / Infra）

### 3.2 节点标准化与去重
对同义别名、编码差异、格式异常进行统一处理，建立唯一标识（canonical ID），降低图谱冗余。

### 3.3 节点富化（Enrichment）
针对每个节点补充上下文特征：

- 时间维度：首次出现、活跃窗口、衰减周期
- 语义维度：标签、行业、地域、战术阶段（可映射 ATT&CK）
- 关联维度：共同出现关系、上下游基础设施关系
- 置信维度：来源可信度、证据强度、冲突评分

### 3.4 威胁知识图谱构建
将富化后的节点写入图谱，形成实体-关系-事件结构，支持：

- 邻域检索（N-hop）
- 路径发现（attack path discovery）
- 社区聚类（campaign clustering）
- 可疑链路发现（suspicious relation discovery）

### 3.5 双模型协同推理
通过两类模型协同提升鲁棒性：

- **模型 1（结构/规则导向）**：强调高精度关系约束与可解释性；
- **模型 2（语义/上下文导向）**：强调弱信号补全与泛化能力。

协同策略可采用（按实现选择）：

- 置信加权融合（weighted ensemble）
- 分歧仲裁（disagreement resolution）
- 级联推理（coarse-to-fine）
- 互蒸馏（mutual distillation）

输出为威胁关联评分、候选关系及解释证据。

## 4. 技术路线 B：多智能体深度抽取与增量学习闭环

### 4.1 多智能体深度抽取
构建多角色 Agent 协作流水线，例如：

- **解析 Agent**：文本切分、段落语义定位、IOC 粗抽取
- **实体 Agent**：实体归类与别名对齐
- **关系 Agent**：关系判定与方向识别
- **验证 Agent**：冲突检测、证据一致性校验
- **融合 Agent**：结果合并、置信评分与最终裁决

该机制有助于降低单模型单次抽取偏差，提高复杂报告场景的稳定性。

### 4.2 增量图谱更新
对新增节点与关系执行增量写入，而非全量重建，重点包括：

- 新增实体插入
- 旧实体属性更新
- 关系权重重估
- 冲突关系降权或隔离
- 历史版本追踪（可选）

### 4.3 增量模型训练
基于新增样本进行持续训练，保留历史能力并适应新威胁分布：

- 增量样本筛选与回放（replay buffer）
- 灾难性遗忘抑制（regularization / replay）
- 周期性评估与回滚策略
- 线上/离线指标联动监控

最终形成“数据来一批，图谱更新一批，模型进化一轮”的持续学习闭环。

## 7. 关键创新点（可用于论文/项目答辩）

1. **双路线协同**：在线情报快速接入 + 增量深抽长期演化；
2. **双模型推理**：结构约束与语义泛化互补；
3. **多智能体抽取**：分工协作降低复杂文本误抽率；
4. **图谱增量更新机制**：避免全量重建，提升时效与可维护性；
5. **增量训练闭环**：持续适配新威胁分布，缓解模型漂移。

## 8. 评估建议（推荐纳入实验）

### 8.1 抽取层
- 实体识别：Precision / Recall / F1
- 关系抽取：Micro-F1 / Macro-F1
- 归一化准确率（Entity Linking Accuracy）

### 8.2 图谱层
- 新增关系正确率（Relation Validity）
- 冲突率（Conflict Ratio）
- 图谱连通性/稠密度变化
- 增量更新时延（Update Latency）

### 8.3 推理层
- Top-K 命中率（Hit@K）
- 路径推理准确率
- 置信校准误差（ECE）
- 双模型协同收益（相对单模型提升）

### 8.4 持续学习层
- 增量阶段性能保持率
- 灾难性遗忘指标
- 每轮增量训练开销（时间/显存/算力）
